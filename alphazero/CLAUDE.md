# AlphaZero Connect Four

## このプロジェクトの位置づけ

```
4moku（DQNの限界を体感）
  → 3moku（DQNが通用する範囲を確認）
  → alphazero（DQNの限界を超える手法を学ぶ）← ここ
```

4mokuで11ステージにわたりDQNの限界を体感した。AlphaZeroは「なぜDQNでは解決できなかった問題を解決できるのか」を理解するための実装。

## DQN vs AlphaZero：根本的な違い

### 1手の決め方

| | DQN | AlphaZero |
|---|---|---|
| **入力** | 盤面 (state) | 盤面 (state) |
| **処理** | NN一発で Q値を計算 → argmax | MCTSで50回シミュレーション → 訪問回数で選択 |
| **先読み** | なし（Q値は間接的な将来予測だが、実際には盤面を見て即決） | あり（MCTSが実際に数手先まで手を打ってみる） |
| **推論速度** | <1ms | 〜1秒（50シミュレーション） |

**ポイント**: DQNは「この盤面の各列のQ値はいくつか？」を即座に答えるが、先読みはしない。AlphaZeroは「この手を打ったら相手はこう返して、それに対してこう打って...」と木を辿って探索する。

### 学習データの作り方

| | DQN | AlphaZero |
|---|---|---|
| **対戦相手** | 外部（ルールベースAI等） | 自分自身（self-play） |
| **データ形式** | (s, a, r, s') — 1手ごと | (s, π, z) — 1ゲーム全体 |
| **教師信号** | 報酬 r と Q値の差（TD誤差） | MCTSの行動確率 π + 勝敗結果 z |
| **問題点** | 相手の強さに学習が制約される | 自分が強くなれば相手（=自分）も自動的に強くなる |

**ポイント**: DQNは「報酬の伝搬」で学習するため、勝敗報酬が最終手にしか付かないConnect Fourでは中盤の因果関係が伝わりにくかった。AlphaZeroは「MCTSが出した行動確率」をそのまま教師にするため、各手に直接の教師信号がある。

### NNの構造

| | DQN | AlphaZero |
|---|---|---|
| **出力** | Q値 (7次元) — 各列の価値 | Policy (7次元) + Value (1次元) |
| **Policy** | なし（Q値のargmaxが方策） | 各列に打つ確率 |
| **Value** | Q値が間接的に価値を表現 | 勝率予測（-1.0〜+1.0） |

**ポイント**: DQNは「この列に打ったらどれくらい得か（Q値）」を1つの数値で表現しようとする。AlphaZeroは「どこに打つべきか（Policy）」と「この局面はどちらが有利か（Value）」を分離して学習する。

## DQNの3つの限界とAlphaZeroの解決策

### 限界1: マルコフ性の仮定（相手を区別できない）

**DQNの問題**: 「盤面だけ」を見て手を決めるため、相手がランダムか強いかを区別できない。同じ盤面でも相手によって最適な手が異なるのに、1つの方策しか持てない。

**AlphaZeroの解決**: MCTSは「相手も最善手を打つ前提」で探索するため、相手の戦略を仮定する必要がない。ミニマックス的に最善手を探すので、相手に依存しない方策を自然に学ぶ。

### 限界2: 信号の薄さ（中盤の因果関係が伝わらない）

**DQNの問題**: Connect Fourは約20手のゲーム。勝敗報酬（±1.0）は最終手にのみ付与され、10手前の「防御をサボった」との因果関係はγの累乗で大幅に減衰。中間報酬を入れると今度はそちらに最適化されてしまう。

**AlphaZeroの解決**: 各手に対してMCTSの行動確率πが直接の教師信号になる。「この局面ではこう打つべき」という情報が中盤の手にもダイレクトに伝わる。

### 限界3: Self-playの不安定性

**DQNの問題**: DQN同士のself-playは破滅的忘却を起こした（ステージ3）。弱い方策同士が「対DQN戦略」に特化し、汎用性を失う。

**AlphaZeroの解決**: MCTSがあるため、NNが弱くても木探索でそこそこ合理的な手を打てる。NNの改善は「MCTSの初手（prior）を良くする」効果があり、さらに探索精度が上がる → 正のフィードバックループ。

## MCTSの仕組み（図解）

### 4ステップの繰り返し

```
1. Select（選択）
   ルートから UCB スコアが最大の子ノードを辿り、葉ノードまで降りる。

   UCB(s,a) = Q(s,a) + c_puct × P(s,a) × √N(s) / (1 + N(s,a))
              ~~~~~~   ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
              活用        探索（訪問が少ない＆NNが推す手を優先）

2. Expand（展開）
   葉ノードで NN を呼び出す → policy, value を取得
   合法手それぞれに子ノードを作成（priorをpolicyから設定）

3. Evaluate（評価）
   NNのvalue予測を使う（ランダムロールアウトは行わない ← これがAlphaZeroの特徴）
   ※ 伝統的MCTSはランダムにゲーム終了まで打って勝敗を確認するが、
     AlphaZeroではNNがその代わりをする

4. Backup（逆伝播）
   葉からルートまで value を伝播。手番が交互なので毎回符号反転。
   各ノードの訪問回数 N と 累積value Q を更新。
```

### 具体例

```
現在の盤面（PLAYER1のターン）:

 0 1 2 3 4 5 6
|. . . . . . .|
|. . . . . . .|
|. . . . . . .|
|. . . X . . .|
|. . O X . . .|
|. O X O . . .|

MCTSがシミュレーション50回を実行:
  列0: 2回訪問, 平均value -0.3
  列1: 3回訪問, 平均value +0.1
  列2: 5回訪問, 平均value +0.2
  列3: 25回訪問, 平均value +0.6  ← 最多訪問（3連が伸ばせる）
  列4: 10回訪問, 平均value +0.4
  列5: 3回訪問, 平均value +0.0
  列6: 2回訪問, 平均value -0.2

行動確率 π: [0.04, 0.06, 0.10, 0.50, 0.20, 0.06, 0.04]
→ 列3が選ばれやすい（MCTSが最も有望と判断）
```

### UCBスコアの直感的な理解

```
UCB = Q(活用) + 探索ボーナス

活用: Q(s,a) = これまでの平均的な評価値
  → 「過去のシミュレーションで良い結果だった手」を優先

探索: c_puct × P(s,a) × √N(parent) / (1 + N(s,a))
  → P(s,a): NNが「良い手」と予測した手を優先（事前確率）
  → √N(parent) / (1+N(s,a)): 訪問回数が少ない手を優先（未探索手の発見）
  → 訪問を重ねるとボーナスが小さくなり、Qの影響が増す

初期: NNの予測（P）に従って探索
中期: 有望な手を深堀り（Qが高い手に集中）
終期: 確信を持った手に訪問が集中 → 訪問回数 ≈ 行動の「推薦度」
```

## AlphaZeroの学習ループ

```
┌─────────────────────────────────────────────┐
│  1. Self-Play（データ生成）                     │
│    - 現在のNNでMCTSを動かし、自分自身と対戦     │
│    - 各手で (state, π) を記録                   │
│    - ゲーム終了後、全手に勝敗結果 z を付与       │
│    - 10ゲーム → 約200手分のデータ               │
└──────────────────┬──────────────────────────┘
                   ↓
┌─────────────────────────────────────────────┐
│  2. Train（NN更新）                             │
│    - バッファからランダムサンプリング              │
│    - Policy loss: -π・log(p)  ← MCTSの判断を模倣│
│    - Value loss:  (v - z)²    ← 勝率予測を改善   │
│    - 10ステップのミニバッチ更新                   │
└──────────────────┬──────────────────────────┘
                   ↓
┌─────────────────────────────────────────────┐
│  3. Evaluate（評価）                            │
│    - vs ランダムAI、vs ルールベースAI で対戦     │
│    - ベスト更新時にスナップショット保存           │
└──────────────────┬──────────────────────────┘
                   ↓
              1に戻る（100イテレーション）
```

### DQNの学習ループとの対比

```
【DQN】
  for each episode:
    for each step:
      action = ε-greedy(Q(state))
      reward = env.step(action)
      buffer.add(s, a, r, s')
      Q学習: Q(s,a) → r + γ*max Q(s',a')

  → 1手ごとに学習、報酬は即時
  → 先読みなし、1手先のQ値予測のみ

【AlphaZero】
  for each iteration:
    // データ生成
    for each game:
      for each move:
        π = MCTS.search(state)  ← ここで50回先読み
        action = sample(π)
      z = 勝敗結果
      buffer.add(state, π, z)

    // 学習
    loss = -π・log(p) + (v-z)²
    optimizer.step()

  → 1ゲーム単位でデータ生成、各手に50回の先読み付き
  → 教師信号はMCTSの探索結果 + 実際の勝敗
```

## 学習ログ

### ステージ1: 初期設定での100イテレーション（2026-03-20）

**設定**: games-per-iter=10, train-steps=10, num-simulations=50, batch-size=128

**結果**:
```
Iter  1: vs Random  3.3%, vs RuleBased 0.0%
Iter 50: vs Random 20.0%, vs RuleBased 0.0%
Iter100: vs Random 23.3%, vs RuleBased 0.0%（100イテレーション通して1勝もなし）
P-Loss: 1.67 → 1.41（改善）
V-Loss: 0.29 → 0.43（悪化）
```

**問題**: vs Random 23%止まり、vs RuleBased 0%。P-Lossは下がる（NNはMCTSの手を模倣できるようになった）が、肝心のMCTSの探索自体がNNの弱さに引きずられ、質の低いデータしか生成できない。正のフィードバックループが回っていない。

**原因分析**:
1. **データ多様性不足**: 10ゲーム/iter × 約35手 = 約350手/iter。Connect Fourの局面空間（~10^12）に対してあまりに少ない。本家AlphaZeroは25,000ゲーム/iterで2,500倍の差
2. **学習不足**: train-steps=10 × batch-size=128 = 1,280手/iter しか学習しない。バッファに35,000手あるのに3.7%しか使っていない
3. **V-Lossの悪化**: 少ないデータで勝率予測を学ぼうとするが、パターンが偏りすぎて汎化できない

**教訓**: AlphaZeroの正のフィードバックループ（NN改善→MCTS改善→良いデータ→NN改善）を回すには、1イテレーションあたりのデータ量が臨界量を超える必要がある。10ゲーム/iterはその閾値を下回っていた。

---

### ステージ2: パラメータ増強（2026-03-20〜、進行中）

**変更**: games-per-iter=10→**200**（20倍）、train-steps=10→**100**（10倍）

**変更の理由**:
- games-per-iter=200: 200ゲーム × 35手 = 7,000手/iter。多様な局面をNNに見せ、正のフィードバックループを回す
- train-steps=100: 128 × 100 = 12,800手/iter を学習。バッファのデータを十分活用する

**期待**: vs Random 70-90%、vs RuleBased 10-30%。DQNのベスト（vs RuleBased 16%）超えが目標。

**所要時間**: 1イテレーション約20-25分、100イテレーションで約35-40時間

**インフラ改善**: チェックポイント機能を追加。5イテレーションごとにNN重み・optimizer状態・バッファを丸ごと保存。`--resume`で途中再開可能（PCスリープ対策）。

## 参考: なぜSelf-playが安定するのか

DQNのself-play（ステージ3）は崩壊した。AlphaZeroでは安定する理由:

1. **MCTSによる品質保証**: NNが弱くても、MCTSが50回の探索で手を補正する。完全にランダムな手は打たない。
2. **教師信号の質**: DQNは「TD誤差」（Q値の差）で学ぶが、これはNN自身の予測に依存するため不安定。AlphaZeroは「MCTSの探索結果」という外部の教師信号で学ぶため、NNの品質に過度に依存しない。
3. **正のフィードバックループ**: NNが改善 → MCTSの初手が改善 → 探索精度向上 → さらに良い教師データ → NNがさらに改善
