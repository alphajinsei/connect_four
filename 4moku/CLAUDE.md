# Connect Four 強化学習プロジェクト

## ユーザーの目標
強化学習（DQN）を勉強中。Connect Four（四目並べ）を題材に、AIが学習して強くなる過程を実感したい。
**最終目標: 初心者の人間より強くなる**

## 学習ロードマップ（現行）

### 現在の方針（ステージ11: 2026-03-19 開始）
**CNN + vs ルールベースAI固定 + 中間報酬なし（勝敗±1.0のみ）+ 遅延コールバック方式**

ステージ10のカリキュラム学習を廃止し、**ルールベースAI固定で直接学習**に方針転換。

理由:
1. **カリキュラム学習は「段階的強化」ではなく「方策の上書き」だった** — ランダムAI向けの方策とルールベースAI向けの方策は根本的に異なる（攻め重視 vs 防御重視）。DQNは1つの方策しか持てないため、noise を下げるたびに前フェーズで学んだ方策が書き換えられ、「段階的に強くなる」のではなく「別の相手向けに再学習する」だけだった
2. **ステージ4の失敗は別の原因だった** — 当時はnext_stateバグ + MLP構造が原因。CNN + バグ修正済みの今なら、ルールベースAI固定でも学習できる可能性がある
3. **ステージ10カリキュラムの結果**: Phase 1→5まで進んだが、Phase 5でvs RuleBased 16%が最高、その後0%に逆戻り。カリキュラムを経ても結局ルールベースAI相手では学べなかった

学習方法:
- ステージ10のep11000スナップショット（vs RuleBased 16%、全体ベスト）から再開
- `--start-phase 5 --no-buffer` でルールベースAI固定学習

## 設計の重要ポイント

### env/connect4_env.py
- `step()` は相手を呼び出さない（self-playやカリキュラム学習に対応するため）
- `get_state(perspective)` で視点を切り替えられる（DQN訓練・self-playで重要）
- 報酬は常に PLAYER1 視点で返し、`GameRunner` が各エージェント視点に変換する
- 状態は shape `(3, 6, 7)` の float32 配列
  - ch0: 自分のコマ
  - ch1: 相手のコマ
  - ch2: 自分のターンなら 1.0、そうでなければ 0.0

### agents/dqn_agent.py
- 入力: `(3, 6, 7)` の3チャンネル盤面、出力: `7`（列インデックス）
- アーキテクチャ（ステージ10〜, CNN-A）: `Conv2d(3→32, 3×3) → ReLU → Conv2d(32→64, 3×3) → ReLU → flatten(2688) → Linear(2688→256) → ReLU → Linear(256→7)`
- パラメータ数: 約710K（旧MLP: 100K）
- **PyTorchベース**（ステージ8で移行）
- 旧MLP版: `Linear(126→256) → ReLU → Linear(256→256) → ReLU → Linear(256→7)`（ステージ8〜9で使用、ステージ10で廃止）
- ε-greedy + 無効手マスキング、ReplayBuffer、ターゲットネットワーク付き
- `save()`/`load()`: 重みのみ（`.pt` 形式、WebUI・スナップショット用）
- `save_checkpoint()`/`load_checkpoint()`: 重み + ReplayBuffer + ε + total_steps + optimizer状態（学習再開用）
  - チェックポイントは `_checkpoint.pt` として別ファイル保存
  - `load_checkpoint(path, load_buffer=False)` でバッファのみスキップ可能
- **旧NumPy版の重み（`.npz`）とは互換性なし** — ステージ8以降はゼロから学習

### agents/rule_based_agent.py
- 学習なし・手書きルールで動く
- 優先順位: ①自分が即勝てる列 → ②相手が次に勝てる列を防ぐ → ③中央寄りをランダム選択
- ランダムAI相手の勝率: 先手99.5%、後手96%
- 現在のDQNの勝率: 未計測（カリキュラム学習で段階的に目指す）

### agents/noisy_rule_based_agent.py（★新規 2026-03-16）
- `RuleBasedAgent` を継承し、`noise` 確率でランダム手を混入
- `noise=0.0`: 完全ルールベースAI、`noise=1.0`: 完全ランダム
- カリキュラム学習の各フェーズで使用（`train.py` が自動切り替え）

### 現在のハイパーパラメータ（train.py）
| パラメータ | 値 | 備考 |
|---|---|---|
| `lr` | 5e-4 | 1e-3では崩壊が激しかったため下げた |
| `epsilon_end` | 0.10 | 0.05では固定相手への過学習が激しかったため上げた |
| `epsilon_decay` | 0.99990 | ~30000ステップ(≒3750ep)でε=0.10に到達 |

| `target_update_interval` | 500 | 200では不安定だったため増やした |

| `batch_size` | 128 | 安定した勾配 |

### GameRunner の遅延コールバック方式（2026-03-17 導入）

**旧方式（廃止）:** 自分が打った直後に `on_step_end(s, a, r, s', done)` を呼んでいた。
- `s'` = 「自分が打った直後の盤面（次は相手のターン）」
- 問題: Q学習の理論的前提が崩れる。`s'` は相手のターンなので、自分がこの状態から行動するわけではない。
  `max Q(s',a')` の予測値は「自分のターンの状態」で学習しているため、「相手のターンの状態」を入力した予測は信頼できない。
- 対症療法として `r - γ*maxQ(s')` と符号反転していたが、本質的には不正確だった。

**現行方式:** `on_step_end` を「相手が1手打った後（再び自分のターン）」に遅延して呼ぶ。
- `s'` = 「相手が打った後、再び自分のターンの盤面」
- 通常のDQN更新式 `Q(s,a) = r + γ * max Q(s',a')` がそのまま使える
- ゲーム終了時は、手を打ったプレイヤーと相手プレイヤーの両方に done=True で確定させる
- 相手側の pending には「中間報酬 + 終局報酬」を合算して返す

### 重要なバグ修正履歴
- **2026-03-16**: `r + γ*maxQ` → `r - γ*maxQ` に修正（当時は next_state が相手ターンだったため）
- **2026-03-17**: GameRunner を遅延コールバック方式に修正し、`r - γ*maxQ` → `r + γ*maxQ` に戻した（next_state が自分のターンになったため、通常の更新式が正しい）
- **2026-03-17**: GameRunner の terminal_reward の符号バグを修正。遅延コールバック導入時から、負けた側に +1.0 が渡されていた（正しくは -1.0）。中間報酬があった頃はノイズに埋もれて発覚しなかったが、中間報酬廃止で顕在化。ステージ5〜7の全学習に影響していた可能性がある

|---|---|---|---|---|
| 5 | 0.80 | ±2.0以上 | 200%以上 | 「3連作り」に最適化、勝ちを逃す |

## 過去の方針と廃止理由（失敗→原因→次の戦略）

このプロジェクトは9回の大きな方針転換を経ている。各失敗から得られた教訓が次の戦略の土台になっている。

### ステージ1・2（ランダムAI相手）→ 失敗

**何をしたか:** DQNをランダムAI相手に学習させた。ステージ2ではカリキュラム的に「過去の自分」を相手にした。

**何が起きたか:** ランダム相手で勝率79%を達成したが、ルールベースAI相手には**勝率0%**。

**推定原因:**
1. **ランダム相手では防御を学べない** — ランダムAIはでたらめに動くので、攻撃を防ぐ動機が生まれない。「適当に並べて4連を狙う」だけで勝てる。
2. **先手・後手の混在** — Connect Fourは先手必勝ゲーム。先手と後手では最適戦略が全く異なるのに、両方を同時に学習させた。
3. **「過去の自分」も弱い** — 弱い相手同士で学習しても強くならない。

**→ 教訓: 「目をつぶった相手に勝つ練習」では実力はつかない。相手には一定の戦略が必要。先手固定で学習すべき。**

### ステージ3（Self-play）→ 失敗

**何をしたか:** DQN同士を対戦させて自己改善を狙った。

**何が起きたか:** 破滅的忘却（Catastrophic Forgetting）が発生。
```
純Self-play: vs Random 77% → 44% に崩壊
```

**推定原因:**
- DQN同士が「対DQN戦略」に特化し、汎用的な戦略を忘れる。
- ランダムAIを混ぜても、「ランダムに勝つ能力」を維持するだけでは意味がない。

**→ 教訓: Self-playは相手の多様性を確保しないと崩壊する。弱い相手同士の学習は退化を招く。**

### ステージ4（vs ルールベースAI固定）→ 失敗

**何をしたか:** 完全なルールベースAI（noise=0.0）を相手に固定して学習。

**何が起きたか:**
```
ep7500: 33%（ベスト） → ep15000: 0% → ep17000: 19% → 不安定に推移
```

**推定原因:**
1. **ε=0の実力は0%だった** — 学習中の評価（ε=0.05混入）では33%に見えたが、純粋推論では0%。εによるランダム手が「まぐれ勝ち」を生んでいた。
2. **固定相手への過学習→崩壊サイクル** — ルールベースAIの「癖」に過学習した後に崩壊するパターンが繰り返された。
3. **相手が強すぎた** — ルールベースAI（vs ランダム勝率99%）は学習初期のDQNには難しすぎ、「負け(-1.0)ばかり」で何も学べなかった。

**→ 教訓: いきなり強い相手と戦わせても信号が薄すぎて学習できない。段階的に強くすべき。固定相手は過学習を招く。**

### ステージ5（NoisyRuleBasedカリキュラム + 大スケール中間報酬）→ 部分的成功 + 構造的バグ発見

**何をしたか:** NoisyRuleBasedAgent（noise=0.8→0.5→0.2）でカリキュラム学習。中間報酬（3連作成+0.50、勝ち手見逃し-0.80など）を導入。Phase 4でスナップショットプール方式を試みた。

**何が起きたか:**
- Phase 3（noise=0.2）では vs RuleBased 99.5% を達成（最高記録）
- しかし Phase 4 移行後に崩壊:
```
ph4移行直後: vs RuleBased 86〜87% → ep3000〜: 18〜58%（急落） → ep5000〜: 5〜11%（崩壊）
```

**推定原因（3つ、うち2つは構造的バグ）:**

1. **Phase 4のプール方式が本質的に問題** — スナップショット（過去の自分）は常にDQNより弱い。弱い相手に勝てる「攻撃的すぎる戦略」が強化され、対強敵戦略が崩れた。ステージ3のself-play失敗と同じ構造。

2. **【構造的バグ】next_stateが「相手のターンの状態」だった** — GameRunnerが `on_step_end` を「自分が打った直後」に呼んでおり、`next_state` は相手のターンの盤面だった。Q学習では `max Q(s',a')` を計算するが、ネットワークは「自分のターンの状態」で学習しているため、「相手のターンの状態」を入力した予測は信頼できない。対症療法として `r - γ*maxQ(s')` と符号反転していたが、本質的には不正確。Phase 3までは相手の20%ランダム性がこの不正確さを吸収していたが、Phase 4（ほぼ完全ルールベース）ではQ値の不正確さが致命的になった可能性が高い。

3. **【構造的バグ】中間報酬が大きすぎた** — 勝敗報酬（±1.0）に対して中間報酬（3連作成+0.50、見逃し-0.80）が大きすぎた。1ゲーム中に中間報酬が複数回発生するため、累積中間報酬が勝敗報酬を上回りうる。すると「勝つ」より「3連を作り続ける」方がQ値的に得になり、最終盤の勝ち手を逃す行動が強化されうる。

**→ 教訓: two-player ゲームでは next_state の視点が重要。中間報酬は勝敗報酬を超えてはならない。**

### ステージ6（遅延コールバック + 小スケール中間報酬 + 目標80% + プール方式Phase4）→ 方針修正

**何をしたか:** ステージ5のバグ2つを修正（遅延コールバック + 中間報酬縮小）してゼロから再学習。Phase 1目標80%、Phase 4はスナップショットプール方式を維持。

**何が起きたか（Phase 1, ep25500時点）:**
```
vs Noisy(0.8): 73〜80%で振動（目標80%に対して1回Clearのみ、2回連続クリアに到達できず）
vs RuleBased: ベスト24.5%（ep22000）
ε: 0.10（収束済み）
ep19000で1回Clear → その後6500ep（約36分）2回目のClearなし
```

**推定原因:**
1. **Phase 1 目標80%が高すぎた** — noise=0.8（80%ランダム）相手の勝率は73〜80%で自然に振動する。2回連続80%超えは運の要素が大きく、学習の質と関係なく足踏みしてしまう。
2. **Phase 4 プール方式は不要** — ステージ5で既に失敗が確認されており、維持する理由がなかった。

**→ 教訓: ほぼランダムな相手での目標は控えめに。早く次のフェーズに進んで「ルールに従う手」への対応力を磨く方が学びが大きい。スナップショットプール方式は弱い相手との対戦で学習を退化させるため廃止。**

### ステージ7（シンプルカリキュラム + 小スケール中間報酬）→ Phase 2 で停滞

**何をしたか:** ステージ6を修正（目標75%、プール廃止、Phase 5追加）。中間報酬は縮小版（3連+0.10、見逃し-0.15）を維持。

**何が起きたか:**
- Phase 1（noise=0.8）はep2000で突破
- Phase 2（noise=0.5、目標70%）でvs Noisy 45〜55%のまま**42000エピソード以上停滞**
- 平均報酬は0.96前後で高止まり（勝率45%なのに報酬が高い）
- vs RuleBased は10〜26%で振動、改善傾向なし

**推定原因: 中間報酬の累積が勝敗報酬に匹敵**
- 1ゲーム中にDQNの手は約8〜10手。半分の手で中間報酬（平均+0.05）が発生すると、1ゲームの累積中間報酬は+0.20〜0.25
- 負けても中間報酬で相殺され、トータル報酬がプラスになりうる
- Q値が「勝つこと」より「3連を作る・防御する」という確実な中間報酬稼ぎに最適化された
- ステージ5で廃止した「中間報酬過大」問題が、スケール1/5でも構造的に残っていた

**→ 教訓: 中間報酬は「毎手蓄積される」ため、1回あたりが小さくても累積で勝敗報酬に匹敵しうる。中間報酬の「1回の値」ではなく「1ゲームの累積値」で勝敗報酬と比較すべき。**

### ステージ8（カリキュラム + 中間報酬なし）→ Phase 2 で停滞

**何をしたか:** ステージ7の方針から中間報酬を完全廃止。勝敗報酬（±1.0）のみでPhase 1からゼロ学習。カリキュラム・昇格条件はステージ7と同一。

**何が起きたか:**
- Phase 1（noise=0.8）はep8000で突破（2回連続クリア）
- Phase 2（noise=0.5、目標70%）で **30000エピソード以上停滞**
```
ep9000〜40000: vs Noisy(0.5) 42〜58%で完全プラトー（ベスト58.5%、目標70%に遠く及ばず）
vs RuleBased: 全期間を通じて0%（ep28000で1%が出たのみ）
平均報酬: -0.07〜-0.20で振動（勝率45%前後を反映した妥当な値）
```

**推定原因: 勝敗報酬だけでは中盤の因果関係が伝搬しない**
1. **信号の薄さ**: Connect Fourは1ゲーム約15〜20手。勝敗報酬は最終手にのみ付与され、途中の「防御をサボった」「攻めを逃した」との因果関係はγの累乗（γ=0.99で10手前は0.90倍）で減衰しながらしか伝搬しない
2. **防御の概念がゼロ**: Phase 1で学んだ「攻撃だけ戦略」は80%ランダム相手には通用するが、50%ルールベース相手は半分の確率で勝ち手を打つ/防御するため、攻撃だけでは勝てない。しかし勝敗報酬だけでは「なぜ負けたか（防御をサボったから）」を直接学べない
3. **ステージ7との対比**: ステージ7は中間報酬の歪みでPhase 2に停滞したが、平均報酬は0.96（中間報酬で嵩上げ）。ステージ8は平均報酬-0.1前後で健全だが、勝率が上がらない。問題の構造が異なる

**→ 教訓: 中間報酬が「多すぎる」のも「ゼロ」も問題。勝敗報酬だけではConnect Fourのゲーム長に対して信号が薄すぎ、特に防御の学習が困難。中間報酬は必要だが、1ゲーム累積が勝敗報酬の1/10以下になるよう極小スケールにすべき。**

### ステージ9（カリキュラム + 極小中間報酬, MLP）→ Phase 1 突破に32000ep + Phase 2 停滞

**何をしたか:** ステージ8の「中間報酬ゼロ」では防御を学べなかったため、極小中間報酬を再導入。3連作成+0.03、3連防御+0.02、見逃し-0.05。2連報酬は廃止。カリキュラム・昇格条件は変更なし。

**何が起きたか:**
- Phase 1で激しい振動を経て、ep32000でようやく突破（ステージ8はep8000で突破 → **4倍遅い**）
- Phase 2（noise=0.5、目標70%）で18000ep停滞、vs Noisy 39〜57%、vs RuleBased 0%
```
Phase 1:
  ep9000: vs RuleBased 14%、vs Noisy 60%
  ep18000: vs Noisy 69.5%（Phase 1内ベスト付近）
  ep20000: vs RuleBased 19%（全期間ベスト）
  ep20000〜27000: 急激に崩壊（vs Noisy 20〜37%、vs RuleBased 0%）
  ep28000〜30000: 一旦回復（vs Noisy 74%）
  ep31000: Clear 1/2（vs Noisy 76.5%）
  ep32000: Clear 2/2（vs Noisy 85.5%）→ Phase 1 突破
Phase 2:
  ep33000〜50000: vs Noisy 39〜57%で停滞、vs RuleBased 0%
```

**ステージ8 vs ステージ9 の比較:**
| | ステージ8（報酬なし） | ステージ9（極小報酬） |
|---|---|---|
| Phase 1突破 | ep8000 | ep32000（4倍遅い） |
| Phase 2 | 30000ep停滞（42〜58%） | 18000ep停滞（39〜57%） |
| vs RuleBased ベスト | 11% (ep1000) | 19% (ep20000) |
| 学習安定性 | Phase 2で安定的に停滞 | **崩壊→回復→崩壊の激しい振動** |

**推定原因:**
1. **中間報酬がQ値の安定性を壊した**: ステージ8は「低いが安定」だったのに対し、ステージ9は大きく振動（69%→20%→74%→45%）。極小スケールでもQ値の学習を擾乱した
2. **noise=0.8（80%ランダム）相手に対する見逃しペナルティが逆効果**: ほぼランダムな相手に「勝ち手を見逃した」ペナルティを与えても、相手が脅威にならないため「無意味な防御」を強化する（ステージ5の失敗と同構造）
3. **中間報酬チューニングの限界**: 大スケール（ステージ5）→小スケール（ステージ7）→極小スケール（ステージ9）と3回試して全て失敗。中間報酬のスケール調整は本質的な解決策ではない

**→ 教訓: 中間報酬は大きくても小さくても問題を引き起こす。中間報酬で「教える」のではなく、ネットワーク自身がパターンを認識できるようにすべき（MLPの空間認識能力の限界が本質的な問題）。**

### ステージ10（CNN + カリキュラム + 中間報酬なし）→ カリキュラムが「方策の上書き」と判明

**ステージ8・9の失敗から方針を根本的に変更:**

1. **QNetworkをMLPからCNN-Aに変更**: 盤面の空間パターン（3連・防御位置等）を畳み込みで自然に認識
2. **中間報酬は完全廃止**: CNNが空間パターンを認識できれば、勝敗報酬のみで十分なはず
3. カリキュラム・昇格条件・eval設定は変更なし

**Phase 1 初回学習結果（50000ep、目標75%時）:**
- CNNの効果は明確: ep4000で vs RuleBased **53%** を達成（全ステージ通じて最高記録）
- しかしep5000以降は vs RuleBased がほぼ0%に低下（一時的に「刺さる戦略」が上書きされた）
- vs Noisy(0.8) は64〜73%で安定するも、75%を2回連続で超えられず（3回Clear 1/2止まり）
- 真の実力が70%前後のとき、200戦評価で75%を2回連続引く確率が低すぎた

**Phase 1目標を75%→70%に引き下げ（2026-03-18）後、カリキュラム全Phase通過:**
- Phase 1→2→3→4→5 と順調に昇格
- しかしPhase 5（ルールベースAI固定）で vs RuleBased 16% が最高値、その後0%に逆戻り
- カリキュラムを経由してもルールベースAI相手では学べなかった

**ルールベースAI固定で再挑戦（ゼロからではなくep11000スナップショットから）:**
```
ep2000: vs RuleBased 9%
ep11000: vs RuleBased 16%（ベスト）
ep12000〜24000: vs RuleBased 0〜10%で振動、改善傾向なし
```
ステージ4と同じ「一時的に上振れ → 崩壊」パターンが再現。CNN + バグ修正済みでも、ルールベースAI固定は厳しい。

**推定原因と最大の教訓: カリキュラム学習は「方策の上書き」に過ぎなかった**

ステージ1〜10の実験を通じて、以下の構造的問題が明確になった：

1. **ランダムAI向けの最適方策とルールベースAI向けの最適方策は根本的に異なる**
   - ランダム相手: 防御不要、攻撃のみで勝てる
   - ルールベース相手: 防御必須、攻撃だけでは負ける
   - 同じ盤面でも、相手によって最適な手が異なる

2. **DQNは1つの方策しか持てない**（マルコフ性の仮定）
   - 盤面だけを見て手を決めるため、「相手がどういうプレイヤーか」を区別できない
   - カリキュラムでnoiseを下げても、前フェーズで学んだ方策が新しい相手向けに「上書き」されるだけ
   - 「段階的に強くなる」のではなく「別の相手向けに再学習する」ことの繰り返し

3. **カリキュラム学習の根本的限界**
   - カリキュラムが有効なのは「易しい問題と難しい問題の最適方策が同じ方向にある」場合
   - ランダムAIとルールベースAIでは最適方策の方向が異なるため、カリキュラムは機能しない
   - noise=0.5の相手に対する最適方策は「攻撃50% + 防御50%」ではなく、盤面ごとに異なる判断が必要だが、DQNには相手のタイプを見分ける情報がない

**→ 教訓: 対人ゲームにおけるカリキュラム学習は、相手タイプごとの最適方策が異なる場合、「段階的強化」ではなく「方策の上書き」になる。DQN単体では相手を区別する機構がないため、複数タイプの相手に対応する汎用方策を学習することは原理的に困難。**

**CNN-A アーキテクチャ:**
```
Conv2d(3→32, 3×3, pad=1) → ReLU → Conv2d(32→64, 3×3, pad=1) → ReLU
→ flatten(64*6*7=2688) → Linear(2688→256) → ReLU → Linear(256→7)
パラメータ数: 約710K（MLPの約7倍）
```

**CNN導入の根拠:**
- MLPはflatten(3×6×7)=126の数値を入力するため、盤面の空間的な位置関係が完全に失われる
- 「横に3連」「斜めに3連を止められる」を126個の独立した数値の組み合わせから学ぶのは非常に困難
- CNNの3×3フィルタなら、3連・4連パターンの検出を自然に学習できる
- 中間報酬で「教える」代わりに、ネットワーク自身に「発見」させる

**速度への影響（ベンチマーク済み）:**
| | MLP | CNN-A |
|---|---|---|
| batch=128（学習） | 1.3ms | 8.8ms（6.9倍） |
| batch=1（eval/推論） | 0.7ms | 1.5ms（2.3倍） |
| パラメータ数 | 100K | 710K |

学習スループットは低下するが、1エピソードあたりの学習効率向上で補えることを期待。

**設計判断:**
- **γは0.99のまま**: CNN導入とγ変更を同時に行うと原因切り分けができないため、まずCNN単体の効果を確認
- **γの調整は次のステップとして検討**: γ=0.95〜0.97はConnect Fourのゲーム長（15〜20手）に対して妥当な範囲
- **スナップショットは2系統**: `best_ep*` + `phaseup_ph*`
- **チェックポイント保存**: ReplayBuffer + ε + Adam状態を `_checkpoint.pt` に保存・復元可能

## 重要な学び: DQNのマルコフ性の限界と「相手を読む」こと

このプロジェクトを通じて得られた**最も本質的な知見**。

### 観察された現象

ステージ1〜10を通じて、一貫して以下のジレンマが観察された：

- **ランダム相手に最適化すると、ルールベースAI相手の勝率が下がる**
- **ルールベースAI相手に最適化すると、ランダム相手の勝率が下がる**

例えばステージ10では、Phase 1（noise=0.8）で vs Noisy 70%前後に達しても vs RuleBased は0%に低下。カリキュラムでnoise を下げると逆にランダム耐性が失われる。

### なぜ両立できないのか: マルコフ性の仮定

DQNは**盤面（現在の状態）だけを見て手を決める**。同じ盤面でも、相手がランダムなら攻め重視、ルールベースなら防御重視が最適だが、**DQNにはその区別がつかない**。

これはDQN（およびQ学習全般）の前提である**マルコフ性**の限界：

- マルコフ性 = 「現在の状態だけで最適な行動が決まる」
- しかし実際には「相手がどういうプレイヤーか」は過去の手の履歴を見ないと分からない
- つまり**盤面は同じでも、相手によって最適行動が異なる** → 状態が不完全（POMDP: 部分観測マルコフ決定過程）

格ゲーのウメハラ選手が「戦いながら相手の癖を読んでそれを刺しに行く」と言うように、**相手の傾向を推定して戦略を切り替える**ことが対人ゲームでは不可欠。しかし現在のDQNにはその機構がない。

### Connect Fourにおける理論的最適解

Connect Fourは完全情報ゲームであり、ミニマックス最適戦略（相手が最善手を打つ前提での最善手）が存在し、先手必勝が証明されている。このミニマックス最適戦略は相手に依存しない（どんな相手にも最善）。

しかし、DQNが勝敗報酬だけでこの戦略を「発見」するのは極めて困難：
- ミニマックス戦略の発見にはゲーム木の深い探索が必要だが、DQNは1手先の即時報酬 + Q値（間接的な未来予測）しか見ない
- MCTS（モンテカルロ木探索）を組み合わせたAlphaZeroのようなアプローチが必要
- DQN単体では「この相手に対して有効な方策」を1つ学ぶことはできるが、汎用的な最適方策には到達しにくい

### 解決の方向性（発展的な手法）

| アプローチ | 概要 | 特徴 |
|---|---|---|
| **状態に履歴を追加** | 相手の直近N手を入力チャンネルに含める | 最も簡単だが、事前に想定した情報しか使えない |
| **RNN/Transformer** | 過去の手の系列を入力に取り、相手のパターンを暗黙的に学習 | ウメハラ的「相手を読む」に最も近い。AlphaStar等が採用 |
| **Opponent Modeling** | 相手の方策を明示的に推定するモジュールを別途持つ | 学術的にはDRON等の研究がある |
| **メタ学習** | 「新しい相手に数手で適応する能力」自体を学習する（MAML等） | 少数の対戦経験から相手タイプを把握 |
| **MCTS + NN** | モンテカルロ木探索とニューラルネットを組み合わせる（AlphaZero方式） | 相手に依存しないミニマックス最適に近づける |

### このプロジェクトでの位置づけ

DQNの限界を体感したことは、強化学習を深く理解する上で非常に価値がある。教科書的なDQNが「万能ではない」ことを実験で確認し、なぜ限界があるのか（マルコフ性の仮定、単一方策の限界）を理解した上で、AlphaZeroやPOMDP系の手法の動機が自然に見えてくる。

## 今後の展望（2026-03-19 更新）

4mokuでのDQN学習は11ステージの試行錯誤を経て、DQN単体の限界を確認。以降は2つの方向で展開する：

1. **3moku（Connect Three, 5×5盤面）でDQN学習** — 盤面を小さくすることで、DQNが対戦ゲームでどこまで通用するかを確認する。CNN + 中間報酬なしの方針は維持。→ `../3moku/` で実装
2. **AlphaZero方式でConnect Four** — MCTS + ニューラルネットで、DQNの限界（マルコフ性、単一方策）を超える手法を学ぶ。→ `../alphazero/` で実装
