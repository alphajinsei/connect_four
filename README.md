# Connect Four 強化学習プロジェクト

四目並べ（Connect Four）を題材に、DQN（Deep Q-Network）による強化学習を実装・実験するプロジェクトです。

PyTorchなどの深層学習フレームワークを使わず、**NumPyだけで**ニューラルネットの順伝播・逆伝播・Adamオプティマイザを手書きしています。ブラックボックスを排除し、DQNの仕組みを一から理解することを目的としています。

## 概要

- **目的**: 強化学習の基礎（DQN）を、Connect Fourで手を動かしながら学ぶ
- **実装**: PyTorchなし、NumPyのみで手書きニューラルネット
- **進捗**: ステージ1・2完了、ステージ3（Self-play）へ

## 学習ロードマップ

| ステージ | 内容 | 状態 |
|---------|------|------|

## DQNの仕組みと実装

### ゲームの状態表現

盤面を `(3, 6, 7)` の3次元配列で表します：

```
チャンネル0: 自分のコマがある場所 → 1.0、それ以外 → 0.0
チャンネル1: 相手のコマがある場所 → 1.0、それ以外 → 0.0
チャンネル2: 自分のターンなら盤面全体が 1.0、相手のターンなら 0.0
```

「自分・相手」は常に視点を揃えて表現するため、PLAYER1とPLAYER2で同じネットワークが使えます（self-playへの拡張を見据えた設計）。

### ネットワーク構造

```
入力: (3, 6, 7) → flatten → 126次元

Linear(126 → 256)
    ↓ ReLU
Linear(256 → 256)
    ↓ ReLU
Linear(256 → 7)

出力: 各列のQ値（7次元）
```

出力の7つの値が「各列に置いたときの期待価値（Q値）」です。埋まった列は `-inf` でマスクし、有効な列の中から最大Q値の列を選びます。

### Q値とは何か

Q値（行動価値）は「**今この状態でこの行動を取ると、最終的にどれだけ得をするか**」の予測値です。

```
Q(状態, 列0) = 0.3   # 0列目に置いた場合の期待勝利価値
Q(状態, 列1) = 0.8   # 1列目に置いた場合 → これが最大なので1列目を選ぶ
Q(状態, 列2) = 0.1
...
```

学習の目標は、この予測値を「真の価値（ベルマン方程式）」に近づけることです。

### ベルマン方程式とターゲットQ値

Q値の更新式（ベルマン方程式）：

```
Q(s, a) ← r - γ * max Q(s', a')
```

- `r`: 今受け取った報酬（勝ち=1.0、負け=-1.0、その他=0）
- `γ`（gamma=0.99）: 割引率。未来の報酬を少し割り引く
- `s'`: 行動後の次の状態
- `max Q(s', a')`: 次の状態での最大Q値

**このプロジェクトでのポイント**：`s'` は「相手のターン」の状態です。相手が最善手を取ると自分に不利になるため、符号を反転して `r - γ * max Q(s')` としています（通常のDQNの `r + γ * max Q(s')` ではない）。

### Experience Replay（経験再生）

プレイした経験 `(状態, 行動, 報酬, 次状態, 終了)` をバッファに貯め、ランダムサンプリングしてバッチ学習します。

```
メリット:
- 時系列相関を壊して学習を安定させる
- 同じ経験を複数回使い回せる
```

バッファ容量は50,000。古い経験は自動で上書きされます。

### ターゲットネットワーク

Q値の更新に使うターゲットネットワークを本体とは別に持ち、200ステップごとにコピーします。

```
メリット: 目標値が頻繁に変わると学習が不安定になるのを防ぐ
         （動くゴールを追いかけるような状況を避ける）
```

### ε-greedy 探索

```
確率ε   → ランダムに行動（探索）
確率1-ε → Q値最大の行動（活用）
```

学習初期は `ε=1.0`（完全ランダム）から始まり、ステップごとに `ε *= 0.99990` で減衰させ、最終的に `ε=0.05` で安定させます（約3750エピソードで到達）。

### 中間報酬（シェーピング報酬）

通常の報酬設計は「勝ち=+1、負け=-1、それ以外=0」です。しかし1ゲームが20〜30手かかるConnect Fourでは、勝利報酬が序盤の手まで薄まります。

```
手1: 0  手2: 0  手3: 0 ... 手25: +1（勝ち）
→ 手1への信号: 0.99^24 ≈ 0.79（かなり薄い）
```

そこで「良い方向に進んでいる」ことへの補助報酬（シェーピング）を追加：

```
2連（自分のコマが2つ繋がった）→ +0.05
3連（自分のコマが3つ繋がった）→ +0.10
相手の2連・3連を作ってしまった場合は逆符号（-0.05、-0.10）
```

**実装の工夫**：盤面全体をスキャンせず、直前に置いたコマを起点に4方向の連続コマ数を数えるだけにしています。

**効果**：中間報酬なしでの勝率 ~64% → あり ~79%

## ハイパーパラメータ

| パラメータ | 値 | 意味 |
|-----------|-----|------|
| `lr` | 1e-3 | Adamオプティマイザの学習率 |
| `gamma` | 0.99 | 割引率（未来の報酬の重み） |
| `epsilon_start` | 1.0 | 探索率の初期値（完全ランダム） |
| `epsilon_end` | 0.05 | 探索率の最小値 |
| `epsilon_decay` | 0.99990 | 探索率の減衰率（1ステップごと） |
| `batch_size` | 128 | 1回の学習に使うサンプル数 |
| `buffer_capacity` | 50,000 | 経験再生バッファの容量 |
| `warmup_steps` | 2,000 | 学習開始前のバッファ蓄積ステップ数 |
| `target_update_interval` | 200 | ターゲットネットの同期間隔（ステップ） |

## 学習結果

### ステージ1: vs ランダムAI
```
Ep   500:  67%
Ep  1000:  79%
Ep  1500:  80%
Ep  2000:  69%  ← ランダム相手への過学習が始まる
```

### ステージ2: カリキュラム学習
```
Phase 1 (vs ランダム):
  ep2000: 65.6% → 65%達成・Phase2へ移行

Phase 2 (vs 弱いDQN):
  ep2500: 66.2% → 60%達成・Phase3へ移行

Phase 3 (vs 中程度DQN):
  ep3000: 67.6% → 60%達成・全フェーズ完了

フェーズ完了後（相手固定）:
  ep3500: 71.6%  ← ピーク
  ep5000: 61.2%  ← 再び過学習
```

**判明した課題**: フェーズ完了後も相手が固定されるため、ε収束後に再び過学習が始まる。
**解決策**: Self-play（相手=自分自身で常に同レベルの相手と戦い続ける）

## 今後の方針

### ステージ3: Self-play

自分自身と対戦させ続けることで、どんな相手にも汎化できる強さを目指します。`GameRunner(env, dqn_agent, dqn_agent)` で実現可能な設計になっています。

一定間隔でスナップショットを取りながら「現在の自分 vs 少し前の自分」を繰り返すことで、安定した学習を目指します。
