# Connect Four 強化学習プロジェクト

四目並べ（Connect Four）を題材に、DQN（Deep Q-Network）による強化学習を実装・実験するプロジェクトです。

PyTorchなどの深層学習フレームワークを使わず、**NumPyだけで**ニューラルネットの順伝播・逆伝播・Adamオプティマイザを手書きしています。ブラックボックスを排除し、DQNの仕組みを一から理解することを目的としています。

## 概要

- **目的**: 強化学習の基礎（DQN）を、Connect Fourで手を動かしながら学ぶ
- **最終目標**: 初心者の人間より強くなる
- **実装**: PyTorchなし、NumPyのみで手書きニューラルネット
- **現在の方針**: 先手（PLAYER1）固定、NoisyRuleBasedカリキュラム学習 + 遅延コールバック方式

## DQNの仕組みと実装

### ゲームの状態表現

盤面を `(3, 6, 7)` の3次元配列で表します：

```
チャンネル0: 自分のコマがある場所 → 1.0、それ以外 → 0.0
チャンネル1: 相手のコマがある場所 → 1.0、それ以外 → 0.0
チャンネル2: 自分のターンなら盤面全体が 1.0、相手のターンなら 0.0
```

### ネットワーク構造

```
入力: (3, 6, 7) → flatten → 126次元

Linear(126 → 256)
    ↓ ReLU
Linear(256 → 256)
    ↓ ReLU
Linear(256 → 7)

出力: 各列のQ値（7次元）
```

出力の7つの値が「各列に置いたときの期待価値（Q値）」です。埋まった列は `-inf` でマスクし、有効な列の中から最大Q値の列を選びます。

### ベルマン方程式とターゲットQ値

Q値の更新式：

```
Q(s, a) ← r + γ * max Q(s', a')
```

**このプロジェクトでのポイント**：`GameRunner` が遅延コールバック方式を採用し、`on_step_end` を「相手が1手打った後（再び自分のターン）」に呼ぶ。これにより `s'` は常に「自分のターンの状態」になるため、通常のDQN更新式がそのまま使える。

旧実装では `s'` が「相手のターンの状態」だったため `r - γ * max Q(s')` と符号を反転していたが、Q値予測の入力として不適切だった（ネットワークは自分のターンの状態で学習しているため）。

### 中間報酬（シェーピング報酬）

勝ち負けだけでは学習シグナルが薄すぎる（1ゲーム1信号）ため、手ごとに細かく報酬を与えます。

```
自分の3連を作った         → +0.10  ← 攻撃を最重視
自分の2連を作った         → +0.03
相手の3連を防いだ         → +0.06
相手の2連を防いだ         → +0.02
相手の勝ち手を見逃した    → -0.15  ← 防御の本質
```

**勝敗報酬（±1.0）が圧倒的に大きいことが最重要**です。中間報酬はあくまで「ヒント」であり、累積しても勝敗報酬を上回らないスケールに設計しています。攻撃報酬 > 防御報酬の比率は維持しつつ、勝つこと自体がQ値的に最も得になるようにしています。

## ハイパーパラメータ

| パラメータ | 値 | 意味 |
|-----------|-----|------|
| `lr` | 5e-4 | Adamオプティマイザの学習率 |
| `gamma` | 0.99 | 割引率（未来の報酬の重み） |
| `epsilon_start` | 1.0 | 探索率の初期値（完全ランダム） |
| `epsilon_end` | 0.10 | 探索率の最小値（0.05では固定相手への過学習が激しかったため） |
| `epsilon_decay` | 0.99990 | 探索率の減衰率（1ステップごと） |
| `batch_size` | 128 | 1回の学習に使うサンプル数 |
| `buffer_capacity` | 50,000 | 経験再生バッファの容量 |
| `warmup_steps` | 2,000 | 学習開始前のバッファ蓄積ステップ数 |
| `target_update_interval` | 500 | ターゲットネットの同期間隔（ステップ）（200では不安定だったため） |

## 過去の方針と廃止理由

### ステージ1・2: ランダムAI相手の学習が不十分だった理由

ランダムAIは「でたらめに動く相手」であり、戦略を持たない。ランダムAIに勝つには「適当に並べて4連を狙う」だけで十分で、**相手の攻撃を防ぐ動機が生まれない**。

実際に確認したところ、ランダム相手で勝率79%を達成したDQNが、ルールベースAI（必ず勝ち手を打ち、必ず負け手を防ぐ）相手には**勝率0%**だった。

> 例え：「目をつぶっている相手に勝つ練習」を続けても、目が開いた相手に勝てるようにはならない。

また、先手・後手の混在も問題だった。Connect Fourは**先手必勝ゲーム**（完全解析済み）のため先手と後手では最適戦略が全く異なる。両方を同時に学習させると戦略が干渉し、どちらも中途半端になる。

### ステージ3: Self-playが失敗した理由

DQN同士で対戦させると「対DQN戦略」に特化し、ランダムAIへの対処能力まで失う（**破滅的忘却**）:

```
純Self-play: vs Random 77% → 44% に崩壊
```

ランダムAIを混ぜて忘却を防ごうとしたが、そもそも「ランダムAIに勝つ能力」を維持してもゴール（初心者に勝つ）には遠く、根本的な解決にならなかった。

### ステージ4: ルールベースAI固定が失敗した理由

20000epの学習でvs RuleBased勝率を測定した結果：

```
ep7500: 33%（ベスト） → ep15000: 0% → その後も不安定に推移
```

2つの根本的な問題があった：

**問題1: ε=0の実力は0%だった**

学習中の評価（ε=0.05混入）では33%に見えたが、ε=0.0（純粋推論）での評価では**0%**だった。εによる「まぐれ勝ち」が数字を水増ししていた。

**問題2: 固定相手への過学習と崩壊サイクル**

ルールベースAIの「癖」に過学習した後に崩壊するパターンが繰り返され、安定した学習ができなかった。また、ルールベースAIは対ランダムAI勝率99%と強すぎるため、学習初期は「負け(-1.0)ばかり」で何を学べばよいかわからない状態だった。

## 現行方針の設計思想（ステージ5）

**「まず攻め方を覚えてから、防ぎ方を覚える」**

### NoisyRuleBasedAgent（段階的カリキュラム）

`noise`パラメータで相手の強さを連続的に調整：

```
noise=1.0 → ランダムAIと同じ（完全ランダム）
noise=0.5 → 50%はルールに従い、50%はランダム
noise=0.0 → 完全なルールベースAI
```

段階的に相手を強くすることで、弱い時期に適切な難易度の信号を与えられる。

### 昇格条件（まぐれ昇格防止）

各フェーズの昇格条件は「vs Noisy 500戦 × 2回連続クリア」。

旧設計（200戦 × 1回）ではPhase 3を1500epで通過するまぐれ昇格が発生した。500戦で統計的振れ幅を縮小し、2回連続クリアで一時的な上振れを排除する。

### Phase 4: プール方式（実験済み・失敗）

完全ルールベース固定（noise=0.0）では「固定相手の癖への過学習→崩壊」が繰り返された。そこでPhase 4を以下の混合方式で設計・実験した：

```
50%: NoisyRuleBased(noise=0.1)
50%: 過去の自分（スナップショット）
```

**結果: 失敗（ph4移行後に急落・崩壊）**

推定原因:
- スナップショット（過去の自分）はDQNより常に弱いため、弱い相手に勝てる戦略に引っ張られる
- 本質的にはステージ3のself-play失敗と同じ構造
- ph4開始直後に不安定な状態でベスト更新が起き、弱い重みがプールに混入した

### 現行方針: ゼロから再学習（ステージ6）

ステージ5で2つの構造的バグを発見・修正したため、ゼロから再学習する:

1. **GameRunner 遅延コールバック**: `next_state` が「自分のターンの状態」になり、Q学習の理論的前提が正しくなった
2. **中間報酬スケール縮小**: 勝敗報酬（±1.0）が支配的になるよう1/5に縮小

カリキュラム（Phase 1→2→3）を改めて実行し、Phase 3以降の方針は結果を見て判断する。

### 攻防セット中間報酬の意義

攻撃と防御を両方報酬設計に含めることで：
- 攻撃偏重（勝てるけど防御しない）を防ぐ
- 防御偏重（負けないけど勝てない）を防ぐ

ただし勝敗報酬の1/10以下に抑え、「勝つこと」自体が最大の動機になるようにしている。
