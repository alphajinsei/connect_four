# DQN 強化学習 Tips集

本プロジェクトの実装を題材に、DQNの仕組みを日本語で整理したメモ。

---

## 1. QNetworkは何を予測するか

QNetwork は「盤面を入力として、各列に置いたときの累積見込み報酬（Q値）を予測する」ニューラルネットワークである。

- **入力:** 盤面の状態（3×6×7 = 126次元。自分のコマ・相手のコマ・手番情報）
- **出力:** 7個の数値 Q(s,0)〜Q(s,6) — 各列に置いたときのQ値

Q値の意味は「この盤面で列 a に置いたら、**そこからゲーム終了までに得られるトータルの報酬の予測値**」。目先の1手だけでなく、その先ずっと最善手を打ち続けた場合の見込みを含む。

行動選択は単純で、Q値が一番高い列を選べばよい（ε-greedyによる探索を除けば）。

---

## 2. DQNの学習 — 予測データと正解データ

### 一言でまとめると

**DQNでは、「現在の盤面でその手を打ったときの累積見込み報酬」を予測データ、「実際に得られた報酬と、遷移先の盤面における最善手の累積見込み報酬に減衰率をかけたものの合計」を正解データとして学習する。**

### 噛み砕くと

- **予測データ:** 「この盤面でこの列に置いたら、最終的にどれくらい得するか」というネットワークの現時点での見積もり
- **正解データ:** 「実際に置いてみたら即座にこれだけの報酬があった。さらに、その結果たどり着いた盤面での最善手の見込みがこれくらいある。その合計」

数式で書くと：

```
予測値:  Q(s, a)

正解値:  r + γ × max Q_target(s', a')
         ~   ~   ~~~~~~~~~~~~~~~~~~~~~~~
         |   |   遷移先 s' での最善手の累積見込み報酬（TargetNetworkで計算）
         |   減衰率（γ = 0.99。将来の報酬を少し割り引く）
         実際にもらった即時報酬
```

### 教師あり学習との違い

教師あり学習では人間が用意した完全な正解ラベルがある（画像→「猫」など）。
DQNの正解データには「見込み」が含まれており、完全な正解ではない。しかし「実際にもらった報酬」という現実の断片が毎回混ざるので、繰り返すうちに見積もりが現実に近づいていく。

この「自分の予測を使って自分を改善する」手法を**ブートストラップ（bootstrap）**と呼ぶ。

### 具体例

```
状況: 盤面 s で列3に置くことを検討中

今の QNetwork:  Q(s, 3) = 0.2 ← 「列3の累積見込みは 0.2」

実際の経験:     列3に置いたら即時報酬 +0.03 をもらい、盤面 s' になった
TargetNet:     max Q_target(s', a') = 0.5 ← 「s' での最善手の見込みは 0.5」

正解(target):  0.03 + 0.99 × 0.5 = 0.525

誤差:          0.525 - 0.2 = 0.325（けっこうズレてた）
→ この誤差を縮める方向にネットワークの重みを更新
```

## 3. なぜ max であって平均ではないのか

正解値の計算で、次の盤面での全ての手のQ値の**平均**ではなく **max（最大値）** を使う。

```
target = r + γ × max Q_target(s', a')
                 ~~~
                 平均ではなく最大
```

理由は単純で、**自分は最善手を選べる**から。次の盤面 s' に到達したら、7列の中からQ値が最も高い手を選ぶはず。だから「全部の手の平均的な良さ」ではなく「一番良い手を選んだ場合の良さ」が正解として適切。

補足: 平均を取るアプローチも存在し、**Expected SARSA** などと呼ばれる。DQN は「最善手を打つ前提での価値」を学ぶので max を使う。

## 4. TargetNetwork（ターゲットネットワーク）の役割

QNetwork の「少し古いコピー」。本プロジェクトでは 500ステップごとに QNetwork の重みをコピーして更新する。

### なぜ必要か

正解値の計算に QNetwork 自身を使うと、学習するたびに正解値も動いてしまう（追いかけっこ状態）。TargetNetwork は一定期間固定されるため、正解が安定し、学習が収束しやすくなる。

---

## 5. ReplayBuffer（経験再生バッファ）の役割

過去の経験 (s, a, r, s', done) を溜めておく記憶装置。最大50,000件を保持。

### 動き方

「溜めてからまとめて学習」ではなく、**毎手学習するが、毎回過去の経験もランダムに混ぜて学習する**。

```
1手打つたびに:
  ① 今の経験を ReplayBuffer に追加             ← 1件追加
  ② ReplayBuffer からランダムに128件取り出す   ← 過去の経験も混ざる
  ③ その128件でネットワークを1回更新
```

### なぜ「今の1件だけ」で学習しないのか

**理由①: 時系列の相関を壊す**

対局中の連続した盤面はほぼ同じ（1手しか違わない）。似たデータばかりで学習すると、直近の状況に偏って過学習する。ランダムに取り出すことで、序盤・中盤・終盤・勝った試合・負けた試合がバラバラに混ざり、偏りが減る。

**理由②: データ効率**

1回の対局で得た経験を1回しか使わないのはもったいない。ReplayBuffer に残しておけば、同じ経験を何度も学習に再利用できる。

## 6. ε-greedy（イプシロン・グリーディ）

行動選択の方針。「探索（未知の手を試す）」と「活用（今の知識で最善を尽くす）」のバランスを取る。

- 確率 ε でランダムな列を選ぶ（**探索**）
- 確率 1-ε でQ値が最大の列を選ぶ（**活用**）
- ε は学習が進むにつれ 1.0 → 0.10 に減衰する

最初は ε=1.0（完全ランダム）で色々な手を試し、経験を蓄積する。学習が進むと ε が下がり、Q値を信頼して活用する比率が増える。ただし完全に 0 にはせず 0.10 で止める（たまに探索を続けることで、局所的な戦略に固定されるのを防ぐ）。

## 7. 遅延コールバック — 本プロジェクト固有の工夫

### 問題

DQN の学習では `s'`（遷移先の盤面）に対して `max Q(s', a')` を計算する。QNetwork は「**自分のターンの盤面**」で学習しているので、`s'` も自分のターンの盤面でなければならない。

しかし、自分が打った直後の盤面は「**相手のターン**」。この盤面を s' として使うと、QNetwork にとって見慣れない入力になり、Q値の予測が信頼できない。

### 解決策

相手が1手打つまで待ってから、DQNに経験を渡す。

```
遅延なし（旧方式）:
  DQNが列3に置く → s' = 「相手のターンの盤面」 ← QNetworkの前提と不一致

遅延あり（現行方式）:
  DQNが列3に置く → 保留にする
  相手が列5に置く → s' = 「再び自分のターンの盤面」 ← QNetworkの前提と一致！
```

これにより、DQNの更新式 `r + γ × max Q(s', a')` がそのまま正しく使える。
