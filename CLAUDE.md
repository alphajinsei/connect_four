## 設計の重要ポイント

### env/connect4_env.py
- `step()` は相手を呼び出さない（self-playやカリキュラム学習に対応するため）
- `get_state(perspective)` で視点を切り替えられる（DQN訓練・self-playで重要）
- 報酬は常に PLAYER1 視点で返し、`GameRunner` が各エージェント視点に変換する
- 状態は shape `(3, 6, 7)` の float32 配列
  - ch0: 自分のコマ
  - ch1: 相手のコマ
  - ch2: 自分のターンなら 1.0、そうでなければ 0.0

### agents/dqn_agent.py
- 入力: `3 × 6 × 7 = 126`、出力: `7`（列インデックス）
- アーキテクチャ: `Linear(126→256) → ReLU → Linear(256→256) → ReLU → Linear(256→7)`
- NumPyベース（PyTorch不使用）
- ε-greedy + 無効手マスキング、ReplayBuffer、ターゲットネットワーク付き

### agents/rule_based_agent.py
- 学習なし・手書きルールで動く
- 優先順位: ①自分が即勝てる列 → ②相手が次に勝てる列を防ぐ → ③中央寄りをランダム選択
- ランダムAI相手の勝率: 先手99.5%、後手96%
- 現在のDQNの勝率: 0%（← これが学習目標）

### 現在のハイパーパラメータ（train.py）
| パラメータ | 値 | 備考 |
|---|---|---|
| `epsilon_decay` | 0.99990 | ~30000ステップ(≒3750ep)でε=0.05に到達 |
| `warmup_steps` | 2000 | バッファを十分蓄積してから学習開始 |
| `target_update_interval` | 200 | ターゲットネットをより頻繁に更新 |
| `buffer_capacity` | 50000 | 多様な経験を保持 |
| `batch_size` | 128 | 安定した勾配 |

### 重要なバグ修正（2026-03-16）
- `_train_step`のターゲットQ値計算を `r + γ*maxQ` → `r - γ*maxQ` に修正
- **理由**: `next_state` は「相手のターン」の状態。相手が最善手を取ると自分に不利なので、符号を反転して自分にとっての損失として計算する必要があった

### 中間報酬（シェーピング）の追加（2026-03-16）
- `env/connect4_env.py` の `step()` に `_shaping_reward()` を追加
- 直前に置いたコマを起点に連数をカウント（2連=±0.05、3連=±0.1）
- 盤面全スキャンではなく置いたマス起点のみなので高速

## 過去の方針と廃止理由

### ステージ1・2（ランダムAI相手、カリキュラム学習）が失敗した理由

**根本的な問題: ランダム相手に学習しても意味がない**

ランダムAIはでたらめに動くだけで戦略を持たない。ランダムAIに勝てる戦略は：
- 「適当に並べて4連を狙う」だけで十分
- 相手の3連を防ぐ必要がほぼない（ランダムなので4連まで揃えることが少ない）

結果として、DQNは「攻撃的に並べる」ことは学んでも「相手の攻撃を防ぐ」ことを学ばない。
ルールベースAI（必ず勝ち手を打ち、必ず負け手を防ぐ）相手に試したところ、**勝率0%**だった。

**先手・後手の混在問題**

ステージ1・2では DQN が PLAYER1（先手）とPLAYER2（後手）両方で学習していたが、
Connect Four は**先手必勝ゲーム**のため、先手と後手では最適戦略が全く異なる：
- 先手: 中央を取り積極的に4連を狙う
- 後手: 相手の攻撃を防ぎながら反撃の機会を待つ

混在学習では両戦略が干渉し、どちらも中途半端になっていた可能性がある。

**カリキュラム学習の「過去の自分」問題**

カリキュラムの Phase 2・3 では「過去の自分（DQN）」を相手にしたが、
その過去の自分もランダム相手に学習したものであり、**弱い相手同士で学習した**に過ぎなかった。

### ステージ3（Self-play）が失敗した理由

**破滅的忘却（Catastrophic Forgetting）**

Self-playでDQN同士が対戦すると「対DQN戦略」に特化していき、ランダムAIへの対処能力まで失った：
```
純Self-play: vs Random 77% → 44% に崩壊
```

**混合学習（random-mix）も根本解決にならなかった**

ランダムAIをプールに混ぜて忘却を防ごうとしたが、
そもそも「ランダムAIに勝つ能力」を維持してもゴール（初心者の人間に勝つ）には遠い。
ランダムAIは「目をつぶって打つ相手」であり、実力向上の素材として不適切だった。

## 現行方針の論拠

**「いろんな戦略を持ったいろんな相手と戦う中で、誰が相手でも勝てるように成長する」**

これを実現するには、まず**最低限の実力（勝ち手を打つ、負け手を防ぐ）**を身につけることが前提。
ルールベースAIはこの「最低限の実力」を持つ最も簡単な相手であり、
ここに勝てるようになることが初心者の人間に勝つための最初の関門。

現行の設計：
- **先手（PLAYER1）固定**: 一つの役割に特化して強さを磨く
- **ルールベースAIのみ**: 意味のある相手（勝ち手・防御手を打つ）に絞る
- **vs RuleBased勝率を主指標**: ランダムAI勝率ではなく、実力のある相手への勝率で進捗を測る
- **自己ベスト更新時のみスナップショット保存**: 本当に強くなったときだけ記録
